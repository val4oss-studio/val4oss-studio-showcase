# Internationalization (i18n) — Next.js 16+ App Router

Routing and translation strategy for showcase sites using the App Router.
No third-party i18n library — pure Next.js primitives.

---

## How it works

```
Browser (Accept-Language) → proxy.ts → redirect /en or /fr
                                           ↓
                                  app/[locale]/layout.tsx   ← validates locale, sets <html lang>
                                  app/[locale]/page.tsx     ← fetches dictionary, passes to sections
                                  Section components        ← receive dict slice as props
```

Translations are **server-only**: zero JS sent to the client for i18n.

---

## Dependencies

```bash
npm install negotiator @formatjs/intl-localematcher
npm install -D @types/negotiator
```

> **Note:** Do NOT add an `i18n` key to `next.config.ts` — that key is **Pages Router only** and has no effect in App Router.

## File structure

```
src/
├── proxy.ts                        # Locale detection + redirect (replaces middleware.ts in Next.js 16)
├── config/
│   └── locale.ts                   # Single source of truth for all layers
└── app/
    ├── page.tsx                    # Root redirect → /{defaultLocale}
    ├── [locale]/
    │   ├── layout.tsx              # Root layout: validates locale, <html lang>, metadata
    │   └── page.tsx                # Fetches dictionary, composes sections
    └── i18n/
        ├── translations.ts         # getDictionary() + Dictionary type
        └── locales/
            ├── en.json
            └── fr.json
```

> There is **no** `app/layout.tsx`. Omitting it makes `app/[locale]/layout.tsx` the root layout,
> which allows setting `<html lang>` dynamically from the URL segment.

## 1. `src/config/locale.ts` — Source of truth

Imported by **all layers** (proxy, layouts, pages, domain mappers).

```typescript
export type Locale = 'en' | 'fr';

export const locales = ['en', 'fr'] as const satisfies readonly Locale[];

export const defaultLocale: Locale = 'en';

export type LocalizedText = Record<Locale, string>; // for DB-translated fields

export function isValidLocale(value: string): value is Locale {
  return (locales as readonly string[]).includes(value);
}
```

Adding a locale = modify **only this file** + add a JSON file + add to `siteContent` in layout.

## 2. `src/proxy.ts` — Locale detection

> In Next.js 16, `middleware.ts` is deprecated. The file is named `proxy.ts`
> and exports a function named `proxy()`.

```typescript
import { NextResponse } from 'next/server';
import type { NextRequest } from 'next/server';
import { match } from '@formatjs/intl-localematcher';
import Negotiator from 'negotiator';
import { locales, defaultLocale } from '@/config/locale';

function getLocale(request: NextRequest): string {
  const acceptLang = request.headers.get('accept-language') ?? '';
  const languages = new Negotiator({ headers: { 'accept-language': acceptLang } }).languages();
  try {
    return match(languages, [...locales], defaultLocale);
  } catch {
    return defaultLocale;
  }
}

export function proxy(request: NextRequest) {
  const { pathname } = request.nextUrl;

  // Static assets (fonts, images…) are served from public/ without locale prefix — never redirect them
  if (/\.[^/]+$/.test(pathname)) return NextResponse.next();

  const pathnameHasLocale = locales.some(
    (locale) => pathname.startsWith(`/${locale}/`) || pathname === `/${locale}`
  );
  if (pathnameHasLocale) return NextResponse.next();

  const locale = getLocale(request);
  request.nextUrl.pathname = `/${locale}${pathname}`;
  return NextResponse.redirect(request.nextUrl);
}

export const config = {
  matcher: ['/((?!_next/static|_next/image|favicon\\.ico).*)'],
};
```

**Why check for file extensions in the function and not only in the matcher?**
The matcher regex runs at the edge before the function. Moving the static asset check
into the function body makes the intent explicit and avoids regex anchoring bugs
(a subdirectory file like `/fonts/a.ttf` won't match a root-level pattern like `[^/]+\.[^/]+$`).

## 3. `app/[locale]/layout.tsx` — Root layout

```typescript
// generateMetadata runs independently — needs its own guard
export async function generateMetadata({ params }): Promise<Metadata> {
  const { locale } = await params;
  if (!isValidLocale(locale)) return {}; // return empty, not notFound()

  const content = siteContent[locale];
  return {
    title: content.title,
    alternates: {
      canonical: `https://example.com/${locale}`,
      languages: { en: 'https://example.com/en', fr: 'https://example.com/fr' },
    },
    // ...
  };
}

// generateStaticParams pre-renders /en and /fr at build time
export function generateStaticParams() {
  return locales.map((locale) => ({ locale }));
}

export default async function LocaleLayout({ children, params }) {
  const { locale } = await params;
  if (!isValidLocale(locale)) notFound(); // authoritative runtime guard

  return (
    <html lang={locale}>  {/* ← JSX expression, not a string */}
      <body>{children}</body>
    </html>
  );
}
```

**Common mistakes:**

| ❌ Wrong | ✅ Correct | Why |
|---|---|---|
| `lang="{locale}"` | `lang={locale}` | String literal, not JSX expression |
| `url: ".../${locale}"` | `` url: `.../${locale}` `` | Double quotes don't interpolate variables |

## 4. `app/i18n/translations.ts` — Dictionary

```typescript
import 'server-only'; // prevents accidental import in Client Components

export interface Dictionary {
  hero: {
    eyebrow: string[];
    tagline: string;
    taglineAccent: string;
    taglineSuffix: string;
    logoAlt: string;
  };
  about: {
    eyebrow: string;
    title: string;
    subtitle: string;
    cards: Array<{ title: string; body: string }>;
  };
}

const dictionaries: Record<Locale, () => Promise<Dictionary>> = {
  en: () => import('./locales/en.json').then((m) => m.default as Dictionary),
  fr: () => import('./locales/fr.json').then((m) => m.default as Dictionary),
};

export async function getDictionary(locale: Locale): Promise<Dictionary> {
  return dictionaries[locale]();
}
```

`Record<Locale, ...>` ensures TypeScript errors if a locale is added to `config/locale.ts`
but not to `dictionaries`.

## 5. `app/[locale]/page.tsx` — Orchestration

```typescript
export default async function LocalePage({ params }) {
  const { locale } = await params;

  // No isValidLocale check here — layout.tsx is the authority.
  // locale as Locale is safe: if locale were invalid, layout would have called notFound().
  const dict = await getDictionary(locale as Locale);

  return (
    <>
      <HeroSection id="home" dict={dict.hero} />
      <AboutSection id="about" dict={dict.about} />
    </>
  );
}
```

**One `getDictionary` call per page.** Pass slices (`dict.hero`, `dict.about`) to sections,
not the full dictionary.

---

## 6. Section components — Receiving translations

```typescript
// Precise typing: the section only sees its own slice
interface HeroSectionProps {
  id: string;
  dict: Dictionary['hero'];
}

export function HeroSection({ id, dict }: HeroSectionProps) {
  return (
    <section id={id}>
      {dict.eyebrow.map((label, i) => (
        <span key={label}>{label}</span>
      ))}
      <p>{dict.tagline} <span>{dict.taglineAccent}</span></p>
    </section>
  );
}
```

Sections are **Server Components** — they receive `dict` as a plain prop.
They do not call `getDictionary` themselves.

---

## 7. Adding a new locale

1. `src/config/locale.ts` → add to `Locale` type and `locales[]`
2. `app/i18n/locales/` → create `[locale].json` with all keys from `Dictionary`
3. `app/i18n/translations.ts` → add entry to `dictionaries`
4. `app/[locale]/layout.tsx` → add entry to `siteContent`
5. `generateStaticParams` and `proxy.ts` update automatically from `locales[]`

---

## Anti-patterns

| Do NOT | Do instead |
|---|---|
| `i18n` key in `next.config.ts` | Use `[locale]` segment + `proxy.ts` |
| `middleware.ts` / `export function middleware()` | `proxy.ts` / `export function proxy()` (Next.js 16) |
| `getDictionary` inside a section component | Call once in `page.tsx`, pass as prop |
| `isValidLocale + notFound()` in every page | Guard lives in `layout.tsx` only; pages use `locale as Locale` |
| `notFound()` inside `generateMetadata` | Return `{}` for invalid locales |
| `Record<string, ...>` for `siteContent` | `Record<Locale, ...>` for exhaustiveness check |
